Security Assessment - 2048 Score Center

Assessment conducted by Lucy Qin - 4/24/2014

Introduction

This report assesses the web security of Walton Lee's web application, 2048 Game Center. This web application maintains high scores for the 2048 game developed by Gabby INSERTLASTNAMEs and provides an API to submit and retrieve scores. This report exposes the security weaknesses in the web application and potential vulnerabilities.

Methodology

I began assessing the security of this web application using black box testing. Later I looked at the actual code to find other vulnerabilities.

Abstract of Findings

Inputs are not checked when submitted. This web application relies on users properly putting in the correct input in the correct format. This exposes many vulnerabilities because the user can manipulate the inputs and inject malicious code. The information posted can easily be manipulated and since inputs are not checked, this also allows attackers to search the database and potentially crash it. Since the highest vulnerabilities come from improper checks on inputs, the web application can be made more secure by preventing against such malicious inputs.

Issues Found

Cross Site Scripting

This web application is vulnerable malicious inputs or scripts. The web application currently accepts scripts and HTML tags, this opens up the potential for an attacker to inject malicious code or use javascript to manipulate database contents. Attackers may also be able to access data. -Can steal cookies, although the current program doesn't use cookies. -Can redirect the site elsewhere
